工厂模式

发表于 2017-03-27   |   分类于 技术

参考文章：http://www.cnblogs.com/zhangchenliang/p/3700820.html

简单工厂模式

工厂类：核心业务逻辑在这里处理
抽象产品类：一个接口或抽象类
具体产品类：实现接口或继承抽象类，工厂生成的对象就是该角色的实例

工厂方法模式

抽象工厂类：核心
具体工厂类：具体业务逻辑在这里
抽象产品类：定义抽象类
具体产品角色：实现或重写接口或抽象类中的方法

抽象工厂模式

抽象工厂角色：定义接口，返回产品，有多个种类的产品，就分别返回不同种类的产品
具体工厂角色：不同种类的产品分别对应不同的具体工厂，每个具体工厂都继承或实现抽象工厂，每一个具体工厂都代表着一个产品族（例如奔驰SUV和宝马SUV为同一产品族）
抽象产品角色：定义抽象类（产品树）
具体产品角色：实现或重写接口或抽象类中的方法

抽象工厂模式一般用于具有产品树和产品族的场景下
抽象工厂模式的缺点：如果需要增加新的产品树，那么就要新增三个产品类，并且要修改三个工厂类。这样大批量的改动是很丑陋的做法，所以可以用简单工厂配合反射来改进抽象工厂
