Git相关知识

发表于 2017-05-22 更新于 2023-10-19 分类于 技术

1，下载git，mac有两种方式，一种是homebrew,一种是xcode自带的.

2，创建版本库，也就是仓库，新建一个文件夹，git init 初始化.

3，一般而言，项目所在目录即为工作区，工作区有一个隐藏目录.git，是Git的版本库。版本库中最重要的就是stage（或者叫index）的暂存区，还有Git为我们自动创建的第一个分支master，以及指向master的一个指针HEAD。另外还有日志目录logs.

4，git add 操作，将文件修改添加到暂存区；git commit操作，将暂存区的所有内容提交到当前分支.

5，忽略某些文件时，需要编写.gitignore文件，并且需要提交到版本库，还可以对它做版本管理.

命令

​ git add

​ git commit -m “xxx”

​ git status 查看仓库当前的状态

​ git diff 查看文件修改

​ git log [—pretty=oneline] [—abbrev-commit] 查看提交日志

​ git reflog 记录每一次命令

​ git reset -hard | head^ 版本回退到某次提交

​ git checkout - - 把文件在工作区的修改全部撤销

​ git reset HEAD file 将添加到暂存区的修改回退到工作区

​ git rm 删掉文件

​ git remote add origin git@server-name:path/repo-name.git 关联一个远程库

​ git push -u origin master 第一次推送master分支到所有内容

​ git push origin master 推送本地分支master到远程分支origin上

​ git pull <远程主机名> <远程分支名>:<本地分支名>

​ git branch –set-upstream master origin/next

​ git clone 克隆远程仓库到本地 ssh/git 速度较快

​ git checkout -b 创建 + 切换分支

​ git branch 创建分支

​ git checkout 切换分支

​ git branch 查看分支

​ git merge 合并某分支到当前分支

​ git merge —no-ff -m ”xxx” 禁用Fast forward模式进行合并，在历史中显示

​ git branch -d 删除分支

​ git log —graph 查看分支合并图

​ git stash 工作区还未提交，隐藏工作现场

​ git stash list 查看工作现场

​ git stash apply stash@{0} 恢复指定的stash，stash内容并不删除

​ git stash drop 删除工作现场

​ git stash pop 恢复工作现场，同时删除stash内容

​ git branch -D 强行删除一个没有被合并过的分支

​ git remote -v 查看远程库信息，如果没有推送权限，看不到push地址

​ git checkout -b origin/ 在本地创建和远程分支对应的分支

​ git branch —set-upstream origin/ 建立本地分支和远程分支的关联

​ git tag 查看所有标签

​ git show 查看标签详情

​ git tag 为某次提交打标签

​ git tag -a -m “xxx” -a指定标签名，-m指定说明文字

​ git tag -s -m “xxx” -s 用私钥签名一个标签 （需安装gpg）

​ git tag -d 删除标签

​ git push origin 推送标签到远程

​ git push origin —tags 推送全部尚未推送到远程的本地标签

​ git tag -d ; git push origin :refs/tags/ 删除远程标签

​ git config —global alias.st status 配置别名
